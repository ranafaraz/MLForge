# MLForge - Offline Evaluation Results

Generated by `python -m evals.harness` over 8 seeds per regime, no API keys, no downloads. Each protocol's **reported** cross-validation score is compared with the **oracle** accuracy of the shipped model on a large held-out test set drawn from the same generator. The gap is **optimism**.

## Headline (regime `highdim`): the same data, three honesties

| Protocol | Reported | Oracle | Optimism |
|---|---:|---:|---:|
| leaky (preproc on all data + best-of-grid) | 0.871 | 0.715 | +0.155 (±0.036) |
| pipeline (no leak, best-of-grid) | 0.793 | 0.743 | +0.050 (±0.023) |
| **nested (honest)** | 0.755 | 0.743 | +0.013 (±0.035) |

Two effects, cleanly separated:

- **Proper pipelining buys a preprocessing-leakage correction.** Fitting the scaler + feature selector inside each fold instead of on all the data cuts optimism by **+0.105** (leaky +0.155 -> pipeline +0.050).
- **Nested cross-validation buys a selection-bias correction.** Estimating the best-of-grid model with an outer loop instead of reporting its own inflated flat-CV score cuts optimism by a further **+0.038** (pipeline +0.050 -> nested +0.013), landing within ±0.02 of the oracle.

## Control (regime `lowdim`): nothing to leak, so the protocols agree

| Protocol | Reported | Oracle | Optimism |
|---|---:|---:|---:|
| leaky | 0.739 | 0.713 | +0.026 (±0.035) |
| pipeline | 0.737 | 0.720 | +0.017 (±0.034) |
| nested | 0.713 | 0.720 | -0.006 (±0.045) |

With few, all-informative features there is no high-dimensional selection step for leakage to exploit, so even the leaky protocol is honest (optimism +0.026). Leakage needs a wide feature space to bite.

## Null test (regime `null`): random labels, true accuracy is 0.5

| Protocol | Reported | Oracle |
|---|---:|---:|
| leaky | 0.763 | 0.500 |
| pipeline | 0.563 | 0.501 |
| nested | 0.491 | 0.501 |

The labels are independent of the features, so the truth is 0.5. The leaky protocol still reports **0.763** -- a signal manufactured entirely by selecting features on the held-out rows. Nested CV reports **0.491**, refusing to be fooled. This collapse is the proof that the leak, not the data, produced the fake accuracy.

## How to read the columns

`Reported` is what you would publish; `Oracle` is the truth on held-out data; `Optimism = Reported - Oracle`. A trustworthy protocol has optimism near zero on every regime -- only `nested` does.
